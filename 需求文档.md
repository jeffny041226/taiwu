# 斗蛐蛐 — 巅峰至臻对战版 需求文档

## 一、产品概述

### 1.1 产品定位
斗蛐蛐是一款古风蛐蛐格斗游戏，支持双人对战和单人训练。玩家通过收集蛐蛐、挑选出战阵容，在回合制对战中运用策略击败对手。游戏采用水墨古风视觉风格，竖屏适配移动端。

### 1.2 核心体验
- **收集**：通过虫市抽笼获取不同品质的蛐蛐
- **策略**：选择3只蛐蛐出战，回合制选择动作
- **对战**：支持真人对战（房间匹配）和训练模式（AI 对手）
- **竞技**：三局两胜制，局间可更换策略

## 二、用户流程

### 2.1 全局流程

```
首页(大厅) → 开房对战/加入房间/训练 → 房间(蛐蛐选择+准备) → 战斗(回合制) → 结算 → 返回首页
                                                            ↗ 虫市(抽笼获取蛐蛐)
首页(大厅) → 背包(查看/放生蛐蛐)
```

### 2.2 首页（大厅）

**功能**：
- 顶部左侧显示用户昵称和头像
- 顶部右侧显示背包入口
- 中央展示游戏 Logo（呼吸动画）+ 蛐蛐素材图（鸣叫动画+声波）+ 枫叶飘落特效
- 背景使用水墨风竖屏背景图

**操作入口**：
- **开房对战**：创建房间，等待对手加入
- **加入房间**：输入房间号加入对战
- **训练**：与 AI 机器人对战

**前置检查**：
- 蛐蛐数量不足3只时，弹出提示"蛐蛐不足3只，先去虫市寻几只吧"
- 数据加载中时，所有操作按钮禁用

**URL 快速加入**：
- 支持 `?room=XXXXX` 参数，已登录+蛐蛐充足时自动进入房间
- 蛐蛐不足时提示，未登录时引导登录后再自动加入
- 链接进入时不闪现首页，直接显示过渡页后跳转

### 2.3 虫市

**功能**：
- 左侧展示抽笼结果（开笼前显示空状态"开笼觅良虫"+蛐蛐动画）
- 右侧展示付款按钮和概率公示
- 跑马灯滚动展示虚拟用户抽取信息

**抽笼类型**：
- 开1笼：随机获取1只蛐蛐
- 开5笼：随机获取5只蛐蛐
- 开10笼：随机获取10只蛐蛐

**概率体系**：
| 品质 | 概率（近似） |
|------|-------------|
| 普通 (common) | ~48% |
| 稀有 (rare) | ~30% |
| 史诗 (epic) | ~15% |
| 传说 (legendary) | ~7% |

> 概率由 `gacha_weight` 加权随机决定

### 2.4 背包

**功能**：
- 展示用户拥有的所有蛐蛐
- 每只蛐蛐卡片显示：名称、称号、品质标签、属性条(攻/防/速)、特性标签
- 支持放生蛐蛐

## 三、房间与战斗系统

### 3.1 房间机制

**创建房间**：
- 每个用户有固定的5位房间号（基于 UID hash 生成，不变）
- 房间号字符集：`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`（排除易混淆字符）
- 创建后进入等待阶段

**加入房间**：
- 通过房间号加入，仅 `waiting` 阶段可加入
- 房间已满/战斗中/不存在时返回错误提示

**房间状态流转**：
```
waiting → ready → battling → roundEnd → battling → ... → finished
                                        → roundEnd → ... → finished
```

### 3.2 蛐蛐选择（准备阶段）

**规则**：
- 进入房间后，展示玩家拥有的全部蛐蛐
- 玩家必须手动选择3只蛐蛐出战
- 选中的蛐蛐有金色边框 + 编号标记（1/2/3）
- 右上角显示"已选 X/3"
- 未选满3只时"准备开战"按钮禁用
- 点击"准备开战"后锁定选择，等待对手准备

**克制关系提示**：
- 猛击 > 鸣叫 > 虚晃 > 格挡 > 猛击
- 纯文字展示，无图标

### 3.3 战斗系统

#### 3.3.1 回合制动作

每回合玩家选择一个动作：

| 动作 | 效果 | 耐力消耗 |
|------|------|---------|
| **猛击** | 高伤害攻击，克制鸣叫 | 12 |
| **虚晃** | 中伤害攻击，克制格挡 | 8 |
| **格挡** | 减伤+恢复耐力/斗性 | 5 |
| **鸣叫** | 提升斗性+恢复耐力，克制猛击外的攻击 | 3 |

**克制关系**：
- 猛击 vs 鸣叫：伤害 ×1.2
- 猛击 vs 虚晃：伤害 ×1.1
- 虚晃 vs 格挡：伤害 ×1.5（破防）
- 虚晃 vs 鸣叫：伤害 ×1.15
- 格挡 vs 猛击：减伤 60%
- 格挡 vs 虚晃：减伤 40%

#### 3.3.2 伤害公式

**猛击伤害**：
```
baseDmg = atk × 1.6 - def × 0.6 + random(0~3) - 2
```

**虚晃伤害**：
```
baseDmg = atk × 1.0 - def × 0.3 + random(0~3)
```

**斗性加成**（加法，非乘法）：
```
spiritAdvantage = (mySpirit - opSpirit) / 200 × 0.5
damage += damage × spiritAdvantage
```

**怯场效果**（斗性差 > 10 时触发）：
```
damage += damage × 10%
```

**伤害上限**：
- 猛击/虚晃：不超过目标 maxHp 的 35%（保底3回合击杀）
- 虚晃：不超过目标 maxHp 的 30%

#### 3.3.3 蛐蛐属性体系

| 品质 | 数值范围 | 特性 |
|------|---------|------|
| 普通 (common) | 攻击14~19，防御15~20 | steadfast/tenacious/resonant |
| 稀有 (rare) | 攻击12~18，防御12~22 | fierce/resonant/steadfast |
| 史诗 (epic) | 攻击22~24，防御7~8 | swift/cunning |
| 传说 (legendary) | 攻击20~25，防御9~21 | fierce/tenacious/swift/steadfast |

**特性效果**：
| 特性 | 效果 |
|------|------|
| fierce (凶猛) | 攻击力 ×1.15 |
| swift (迅捷) | 速度 ×1.2 |
| cunning (狡黠) | 虚晃伤害 ×1.2 |
| steadfast (坚韧) | 格挡减伤额外 +20% |
| tenacious (顽强) | HP 低于 30% 时防御 ×1.3 |
| resonant (共鸣) | 鸣叫斗性获取 ×1.5 |

#### 3.3.4 战斗流程

1. **开战动画**：全屏暗底 + 金色光芒脉动 + 双方头像 VS 对峙 + "开战"文字缩放入场（2秒）
2. **回合执行**：双方选动作 → 同时结算 → 显示扣血浮动文字 + 战报
3. **蛐蛐击败**：一方蛐蛐 HP 归零 → 局间结算动画
4. **局间结算**：全屏动画显示胜负 + 比分 + 败阵蛐蛐信息 + "下一局"按钮
5. **游戏结束**：一方赢得2局 → 最终结算动画 + "返回大厅"按钮

**自动模式**：
- 战斗中可开启自动出招（系统随机选择动作）
- 开启自动时，局间2.5秒后自动点"下一局"
- 手动点"下一局"后显示"等待对手准备中..."，直到双方都 ready 后才开始

### 3.4 战斗画面 UI

**布局**（竖屏）：
- 顶部：对手信息（头像+昵称+血条+耐力条+蛐蛐名）
- 中间：竞技场沙盘（泥沙径向渐变+金色环框+"斗"字贴图）+ 双方蛐蛐对峙
- 底部：自己信息 + 4个动作按钮 + 自动模式开关

**视觉规范**：
- 背景：#0a0807（暗黑）
- 强调色：#c5a059（金色）
- 血条：暗赭红(#a05040→#5a2218)，低血量变暖褐+呼吸闪烁
- 耐力条：水墨蓝灰(#5a7890→#3a5068)
- 字体：Noto Serif SC（中文衬线）

## 四、WebSocket 协议

### 4.1 连接

- 端点：`/ws/battle`
- 心跳：`ping`/`pong`（30秒间隔）
- 消息格式：`{ type: string, payload: unknown }`

### 4.2 消息类型

| 方向 | 类型 | 说明 |
|------|------|------|
| 客户端→服务端 | `room:create` | 创建房间 |
| 客户端→服务端 | `room:join` | 加入房间（payload 含 roomId） |
| 客户端→服务端 | `room:practice` | 训练模式 |
| 客户端→服务端 | `battle:ready` | 准备开战（payload 含 cricketIds: number[]） |
| 客户端→服务端 | `battle:action` | 选择动作（payload 含 action） |
| 服务端→客户端 | `room:created` | 房间创建成功 |
| 服务端→客户端 | `room:joined` | 加入房间成功 |
| 服务端→客户端 | `room:state` | 房间状态推送 |
| 服务端→客户端 | `room:error` | 错误提示 |
| 服务端→客户端 | `battle:roundResult` | 回合结果 |
| 服务端→客户端 | `battle:roundWin` | 本局胜负 |
| 服务端→客户端 | `battle:gameOver` | 整场结束 |

### 4.3 房间状态结构

```typescript
{
  roomId: string;
  phase: 'waiting' | 'ready' | 'battling' | 'roundEnd' | 'finished';
  leftPlayer: Player | null;   // 房主
  rightPlayer: Player | null;  // 对手/AI
  currentLeftCricket: number;  // 左方当前蛐蛐索引
  currentRightCricket: number; // 右方当前蛐蛐索引
}
```

## 七、非功能性需求

### 7.1 性能
- 首屏图片使用 WebP 格式压缩
- 关键图片预加载（ReactDOM.preload）
- 首屏 LCP 图片加 `fetchpriority="high"`
- 非首屏图片懒加载

### 7.2 兼容性
- 竖屏适配移动端
- 微信内置浏览器兼容
- 不支持横屏模式

### 7.3 安全
- WebSocket 心跳检测（30秒超时）
- 蛐蛐数量服务端校验（<3只禁止参战）
- 房间状态服务端校验（非 waiting 禁止加入）
- 伤害上限服务端强制（防止一击秒杀）

### 7.4 错误处理
- WS 断开时自动标记房间 finished，5秒后清理
- 旧房间残留时自动清理重建
- 局间 ready 重置，防止跳过等待
- 房间不存在时前端显示错误页+返回大厅按钮
