### 素材替换约定

1. **同文件名替换**: 用新素材直接覆盖 `public/assets/` 下的同名文件即可换皮
2. **同尺寸同格式**: 替换的素材必须保持相同的像素尺寸和文件格式
3. **9-slice 图**: 标注了 9-slice 的图片，替换时需保持相同的 border 区域
4. **Lottie 动画**: 替换 JSON 文件时，需保持相同的画布尺寸，否则需同步更新代码中 LottiePlayer 的容器尺寸
5. **音频**: 替换时保持相同的时长范围（循环音频）和码率规格
6. **蛐蛐图片**: 通过 S3 `image_key` 管理，替换只需更新 S3 上的图片文件

### 5.4 WebSocket 服务器

#### 架构

```
Next.js App (port 3000)
  └── src/lib/ws-client.ts        # 客户端 WS 连接

WebSocket Server (port 3001, 独立进程)
  └── server/
      ├── index.ts                # WS 服务入口
      ├── ws-handler.ts           # 连接/消息路由
      ├── room-manager.ts         # 房间 CRUD
      ├── battle-resolver.ts      # 回合计算
      └── ai-opponent.ts          # 训练模式 AI
```

#### 消息流 (已在需求文档中定义)

- 客户端→服务端: `room:create`, `room:join`, `room:practice`, `battle:ready`, `battle:action`
- 服务端→客户端: `room:created`, `room:joined`, `room:state`, `room:error`, `battle:roundResult`, `battle:roundWin`, `battle:gameOver`
- 心跳: `ping`/`pong`, 30s 间隔

#### 服务器实现要点

- 使用 `ws` 库，监听 3001 端口
- 房间数据存内存 `Map<string, Room>`，定期清理已完成/超时房间
- 回合结算逻辑服用 `src/lib/battle-calc.ts`（通过共享模块路径）
- AI 对手: 加权随机选动作（偏好克制对手上回合动作）

### 8.3 技术风险评估

| 风险 | 级别 | 缓解措施 |
|------|------|---------|
| Lottie 动画制作成本高 | 中 | 占位期用 CSS 动画替代，Lottie 后期替换；动画画布尺寸已固定，替换时直接换 JSON |
| 枫叶粒子性能 | 低 | Canvas 2D 最多 15 片，requestAnimationFrame 控制帧率 |
| WS 断线导致状态不一致 | 中 | 心跳检测 30s，断线 5s 清理房间，服务端自动标记 finished |
| 微信浏览器兼容 | 中 | 避免使用 WebP（降级 PNG），避免 ES2020+ 特性（polyfill），测试微信内置浏览器 |
| 蛐蛐图片体积大 | 低 | S3 + CDN 分发，本地缓存，卡片用 200×200 缩略图 |
