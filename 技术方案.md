# 斗蛐蛐 — 技术方案与架构设计

## 一、技术栈

| 层 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Next.js 16 (App Router) | 7 个页面路由，全部 `use client` |
| 后端框架 | Express + 原生 `ws` | 单进程合并 HTTP(4000) + WebSocket |
| 语言 | TypeScript | monorepo 三包共享类型 |
| 包管理 | pnpm workspace | `@taiwu/backend` / `@taiwu/shared` / 根(frontend) |
| 数据库 | Supabase (PostgreSQL) | 懒加载，无 DB 时走内存 fallback |
| 鉴权 | 山海 Passport API | 手机号+短信验证码 |
| 支付 | 微信支付 V3 (服务商 H5) | Mock/Real 双模式 |
| 战斗引擎 | 纯函数共享 | `packages/shared/src/lib/battle-calc.ts` |
| 样式 | Tailwind CSS 4 + CSS 变量 | 古风暗金色系 |
| 测试 | Vitest | 单元测试 |
| 构建 | pnpm + tsx | 后端 tsx 直接运行 |

## 三、核心流程

### 3.1 登录流程

**关键设计**：
- **万能验证码 666666**：测试环境跳过 Passport，生成 `shanhai-{mobile}` 本地 UID
- **Passport API 不可用**：`send-code` 返回 mock 标记，前端仍可用 666666 登录
- **Token 缓存**：5 分钟内存 TTL，避免每次请求都调 Passport verifyToken
- **新用户赠送**：3 只起始蛐蛐（无赠送抽奖次数），通过 DB 或内存存储

**战斗规则**：
- 克制链：`heavy_strike > chirp > feint > block > heavy_strike`（三角克制 + 额外克制关系）

- 支持两种模式：tag_team（车轮战，3 只蛐蛐轮流出战）/ best_of_3（三局两胜）

## 四、数据模型

### 4.1 users 表

```sql
uid          VARCHAR PK       -- Passport UID 或 shanhai-{mobile}
nick_name    VARCHAR          -- 昵称
```

```sql
id            SERIAL PK      -- 模板ID (1-20)
name          VARCHAR        -- 名称
title         VARCHAR        -- 称号
tier          VARCHAR        -- 品质: common/rare/epic/legendary
attack        INTEGER        -- 基础攻击
defense       INTEGER        -- 基础防御
speed         INTEGER        -- 基础速度
hp_base       INTEGER        -- 基础HP (默认100)

image_key     TEXT           -- 云端图片URL
```

### 4.3 user_crickets 表

```sql

defense       INTEGER        -- 个体防御

max_hp        INTEGER        -- 个体最大HP
max_stamina   INTEGER        -- 个体最大耐力
```

## 五、关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 鉴权方式 | 山海 Passport | 接入山海平台生态，手机号验证码登录 |
| 支付模式 | 服务商 H5 | 山海平台要求服务商模式，H5 适合手机浏览器场景 |
| DB 策略 | Supabase 懒加载 + 内存fallback | 开发/演示无 DB 可运行，生产环境无缝切换 |
| 后端架构 | 单进程 Express+WS | 小团队轻量部署，避免微服务复杂度 |
| 战斗计算 | 共享纯函数 | 前后端计算结果一致，服务端权威裁决防篡改 |
| Token 缓存 | 内存 Map (5min TTL) | 减少 Passport API 调用，重启后自动失效 |
| 支付 Mock/Real | 自动切换（配置检测） | 开发阶段无证书可跑通流程，上线填入配置即生效 |
| 图片托管 | Supabase Storage | 避免前端打包图片资源，支持动态下发 |
| 设计基线 | 390×844 竖屏 | 以 iPhone 14/15 为基准，仅支持竖屏 |
